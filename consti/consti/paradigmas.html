Paradigmas de Programação

Um paradigma de programação é um modelo que procura a melhor
forma de resolver um problema usando uma determinada linguagem.
Os paradigmas determinam as regras e princípios que orientam o
estilo de programação e a forma de estruturar o código.

Paradigma Imperativo

Este paradigma tem como objetivo explicar como executar passo a passo,
na sequência correta que deve ser realizada a tarefa, ou seja, este
paradigma tem como foco “como” fazer e não “o que” fazer na tarefa.
O paradigma imperativo caracteriza-se pela execução sequencial de instruções
que modificam o estado do programa. Esse modelo é adequado quando se pretende
ter um controlo detalhado sobre o fluxo de execução e sobre as alterações das
variáveis ao longo do tempo.

As linguagens imperativas, ao serem executadas, eventualmente modificam o
estado do programa. Nesse mesmo sentido, a programação imperativa resume-se
a três conceitos:

* Estados de uma máquina abstrata, descritos por valores de um conjunto de variáveis;
* Reconhecedores de estados — expressões compostas por relações entre esses valores
ou os resultados de operações utilizando valores;
* Comandos de atribuição e controle que manipulam o estado.

As características deste são:

* Variáveis: As variáveis mantêm o estado de um programa imperativo e são
associadas com localizações de memória que correspondem a um endereço e um valor
de armazenamento. O valor da variável pode ser acedido e alterado direta ou
indiretamente, através de um comando de atribuição ou de um comando de leitura.
* Atribuições: O comando de atribuição introduz uma dependência de ordem no programa:
o valor de uma variável é diferente antes e depois de um comando de atribuição.
* Sequência: O resultado do processamento de um programa depende da ordem na
qual os comandos são escritos e executados, ou seja, da sequência na qual os
comandos estão escritos.

As linguagens de programação que adotam o paradigma imperativo são
por exemplo: C, Java e Python.
As vantagens incluem eficiência e controlo detalhado sobre o fluxo de
execução, e as desvantagens é ser mais complexo e possíveis efeitos colaterais.

Paradigma Declarativo

A programação declarativa é um paradigma de programação que se concentra
em descrever o que um programa deve realizar, em vez de como deve ser executado.

Existem alguns princípios fundamentais que guiam a programação declarativa.

* O primeiro princípio, o princípio da imutabilidade, sugere que os dados não devem
ser modificados após serem criados.
* O segundo princípio é o da transparência referencial, que determina que uma
função deve sempre retornar o mesmo resultado para os mesmos argumentos.
* O terceiro princípio é o da avaliação preguiçosa, que significa que as
expressões só são avaliadas quando seus resultados são realmente necessários.
Existem várias linguagens de programação que adotam o paradigma declarativo,
a linguagem Prolog e a linguagem Haskell são das mais conhecidas.

A linguagem Prolog baseia-se em lógica de predicados, o programador define
factos e regras lógicas e o sistema de execução é responsável por encontrar
soluções para as consultas feitas ao programa.

A linguagem Haskell baseia-se em programação funcional. Nesta linguagem o
programador define funções puras, que não têm efeitos colaterais, e expressões
que são avaliadas de forma preguiçosa.

As programações declarativas têm como vantagens a redução de efeitos
colaterais, a redução de bugs e maior legibilidade dos códigos, mas também
têm desvantagens como a difícil adaptação dos programadores.

Paradigma Funcional

Este paradigma é baseado em funções matemáticas puras, ou seja,
sempre o mesmo resultado para a mesma entrada e a imutabilidade de dados.

Os programas tornam-se mais confiáveis e previsíveis, reduzindo a probabilidade
de bugs. É utilizado em linguagens como Haskell, JavaScript e Python.

Paradigma Lógico

O paradigma lógico é um estilo de programação declarativa que se baseia
na lógica simbólica e na inferência para resolver problemas.
Neste modelo, o programador estabelece factos e regras lógicas que
representam o conhecimento de um determinado domínio. Linguagens de
programação lógica permitem, assim, que a implementação encontre
automaticamente as sequências de factos e inferências necessárias para
satisfazer os objetivos definidos.

O sistema usa inferência para deduzir soluções.
A linguagem mais comum é o Prolog.

Paradigma Orientado a Objetos

A Programação Orientada a Objetos (POO) é um paradigma de programação
que modela o mundo real por meio de objetos, que são instâncias de classes.
Cada objeto combina atributos, que representam os seus dados, e métodos,
que definem o seu comportamento.

O objeto é uma instância concreta de uma classe, ou seja, uma entidade
que possui valores específicos para os atributos definidos na classe.

A classe é o modelo ou estrutura que define os atributos e
métodos de um tipo de objeto.

Este paradigma baseia-se ainda nos seguintes princípios:

O encapsulamento consiste em proteger os dados internos de um objeto,
permitindo o acesso apenas por meio de métodos específicos.

A herança permite que uma classe herde propriedades e métodos de outra,
promovendo a reutilização de código e facilitando a criação de
hierarquias de classes.

O polimorfismo possibilita que um mesmo método apresente diferentes
comportamentos de acordo com o tipo do objeto que o utiliza.

A abstração consiste em representar apenas os aspetos essenciais
de um objeto, ocultando detalhes desnecessários.

Este paradigma é amplamente utilizado no desenvolvimento de sistemas
modernos devido à sua capacidade de representar entidades do mundo real
de forma clara e eficiente.

As linguagens de programação que possibilitam aplicação prática
dos princípios da POO são: Java, C++, Python, Ruby, C#, Swift e PHP
(nas versões modernas).
